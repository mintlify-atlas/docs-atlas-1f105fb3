---
title: "yabridge.toml"
description: "Configure yabridge on a per-plugin basis using a TOML configuration file."
---

Yabridge can be configured on a per-plugin basis by creating a `yabridge.toml` file. This file controls settings like [plugin groups](/configuration/plugin-groups) and [compatibility options](/configuration/compatibility-options) for individual plugins or entire directories of plugins.

## File locations

Place `yabridge.toml` in either the same directory as the bridged plugin `.so` or `.clap` file you want to configure, or in any of its parent directories. Yabridge searches upward through the directory tree and uses the first file it finds.

For most setups, create the file at one of these paths depending on the plugin format:

| Plugin format | Recommended location |
| ------------- | -------------------- |
| VST2 | `~/.vst/yabridge/yabridge.toml` |
| VST3 | `~/.vst3/yabridge/yabridge.toml` |
| CLAP | `~/.clap/yabridge/yabridge.toml` |

## File format

`yabridge.toml` uses the [TOML](https://toml.io) format. The file is made up of _sections_, where each section header is a glob pattern and the keys beneath it are the options to apply.

```toml
["PluginName.so"]
group = "my-group"
frame_rate = 30
```

### How sections are matched

Each section header is a case-sensitive [glob pattern](https://www.man7.org/linux/man-pages/man7/glob.7.html) that matches paths to yabridge `.so` and `.clap` files, relative to the `yabridge.toml` file itself.

<Note>
Only the **first** `yabridge.toml` file found and only the **first** matching glob pattern within that file are used. Later sections that would also match the same plugin are ignored. Yabridge prints the matched file and section on startup so you can verify what configuration is being applied.
</Note>

### Matching directories

Glob patterns can match entire directories instead of individual files. When a pattern matches a directory, the settings in that section apply to all plugins in that directory and any of its subdirectories.

```toml
# Matches all plugins inside the ToneBoosters directory
["ToneBoosters"]
group = "toneboosters"
```

<Note>
For VST3 plugins, you should match on the `.vst3` bundle directory rather than the `.so` file deep within it. For example, use `["FabFilter Pro-Q 3.vst3"]` rather than trying to match the internal `.so` path. Similarly for CLAP plugins, match on the `.clap` file directly.
</Note>

## Examples

### VST2 configuration

Place this file at `~/.vst/yabridge/yabridge.toml`. All paths are relative to this file.

```toml
# ~/.vst/yabridge/yabridge.toml

["FabFilter Pro-Q 3.so"]
group = "fabfilter"

["MeldaProduction/Tools/MMultiAnalyzer.so"]
group = "melda"

# Matches an entire directory and all files inside it, make sure to not include
# a trailing slash
["ToneBoosters"]
group = "toneboosters"

["PSPaudioware"]
editor_coordinate_hack = true

["Analog Lab 3.so"]
editor_xembed = true

["Chromaphone 3.so"]
hide_daw = true

["sforzando VST_x64.so"]
editor_force_dnd = true
frame_rate = 24

["Loopcloud*"]
disable_pipes = true

# Simple glob patterns can be used to avoid unneeded repetition
["iZotope*/Neutron *"]
group = "izotope"

# Since this file has already been matched by the above glob pattern, this won't
# do anything
["iZotope7/Neutron 2 Mix Tap.so"]
group = "This will be ignored!"

# Of course, you can also add multiple plugins to the same group by hand
["iZotope7/Insight 2.so"]
group = "izotope"

# This would cause all plugins to be hosted within a single process. Doing so
# greatly reduces the loading time of individual plugins, with the caveat being
# that plugins are no longer sandboxed from each other.
#
# ["*"]
# group = "all"
```

### VST3 configuration

For VST3 plugins, match on the `.vst3` bundle directory instead of a `.so` file. Place this file at `~/.vst3/yabridge/yabridge.toml`.

```toml
# ~/.vst3/yabridge/yabridge.toml

["FabFilter*.vst3"]
group = "fabfilter"
editor_disable_host_scaling = true

["Chromaphone 3.vst3"]
hide_daw = true

["Misstortion2.vst3"]
editor_disable_host_scaling = true

["*/*Spectral*.vst3"]
vst3_prefer_32bit = true

# These options would be applied to all plugins that do not already have their
# own configuration set
["*"]
editor_force_dnd = true
editor_disable_host_scaling = true
```

### CLAP configuration

With CLAP plugins, match on the Linux `.clap` plugin file directly, the same way you match `.so` files for VST2. Place this file at `~/.clap/yabridge/yabridge.toml`.

```toml
# ~/.clap/yabridge/yabridge.toml

["fb799964.clap"]
hide_daw = true
```
