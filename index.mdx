## What is yabridge?

yabridge lets you run Windows VST2, VST3, and CLAP audio plugins on Linux by hosting them inside Wine while presenting them to your Linux DAW as native plugins. Unlike older solutions, yabridge uses a modern concurrent architecture with Unix domain sockets and shared memory audio buffers, giving you near-native performance.

## Getting started

<Steps>
  <Step title="Install Wine Staging">
    yabridge requires a recent version of Wine Staging. On Debian/Ubuntu, install from the [WineHQ repositories](https://wiki.winehq.org/Download). On Arch/Manjaro, install `wine-staging` from your package manager.
  </Step>
  <Step title="Install yabridge and yabridgectl">
    Install via your distro's package manager (Arch, Fedora, NixOS, OpenSUSE) or download the prebuilt binary archive from [GitHub Releases](https://github.com/robbert-vdh/yabridge/releases) and extract to `~/.local/share`.
  </Step>
  <Step title="Add your plugin directories">
    Tell yabridgectl where your Windows plugins live:
    ```bash
    yabridgectl add "$HOME/.wine/drive_c/Program Files/Steinberg/VstPlugins"
    yabridgectl add "$HOME/.wine/drive_c/Program Files/Common Files/VST3"
    yabridgectl add "$HOME/.wine/drive_c/Program Files/Common Files/CLAP"
    ```
  </Step>
  <Step title="Sync and scan">
    Run `yabridgectl sync` to set up yabridge for all your plugins, then rescan your DAW's plugin library.
    ```bash
    yabridgectl sync
    ```
  </Step>
</Steps>
