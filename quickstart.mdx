---
title: "Quick start"
description: "Install your Windows plugins and get them running in your Linux DAW in a few steps."
---

This guide walks you through the complete workflow for setting up Windows plugins with yabridge, from installing Wine to scanning plugins in your DAW.

<Steps>
  <Step title="Install Wine Staging">
    yabridge requires a recent version of Wine Staging.

    On **Arch** and **Manjaro**:

    ```bash
    sudo pacman -S wine-staging
    ```

    On **Debian**, **Ubuntu**, **Linux Mint**, and **Pop!\_OS**, install from the [WineHQ repositories](https://wiki.winehq.org/Download), as the versions in the distro repositories are often too old.

    On other distros, install `wine-staging` using your package manager.
  </Step>

  <Step title="Install yabridge and yabridgectl">
    Install yabridge and its companion utility yabridgectl. See the [installation guide](/installation) for full per-distro instructions.

    On **Arch** and **Manjaro**:

    ```bash
    sudo pacman -S yabridge yabridgectl
    ```

    On **Ubuntu**, **Debian**, and other distros, download the prebuilt binary archive from the [GitHub releases page](https://github.com/robbert-vdh/yabridge/releases) and extract it to `~/.local/share`:

    ```bash
    tar -C ~/.local/share -xavf yabridge-x.y.z.tar.gz
    ```

    Then add yabridge to your `PATH`:

    ```bash
    echo 'export PATH="$PATH:$HOME/.local/share/yabridge"' >> ~/.bashrc
    ```

    Reopen your terminal after this step.
  </Step>

  <Step title="Install your Windows plugins in Wine">
    Install your Windows VST2, VST3, and CLAP plugins the same way you would on Windows — by running the plugin installers through Wine. Most installers will install to the standard locations inside your Wine prefix automatically:

    - **VST2**: `C:\Program Files\Steinberg\VstPlugins` (sometimes `VSTPlugins` or a directory directly under `Program Files`)
    - **VST3**: `C:\Program Files\Common Files\VST3`
    - **CLAP**: `C:\Program Files\Common Files\CLAP`

    You do not need to do anything special — just run the installers and let them install to the default locations.
  </Step>

  <Step title="Add your plugin directories to yabridgectl">
    Tell yabridgectl where your Windows plugins are installed. Use `yabridgectl add` with the path to each plugin directory inside your Wine prefix.

    For the standard VST2 directory:

    ```bash
    yabridgectl add "$HOME/.wine/drive_c/Program Files/Steinberg/VstPlugins"
    ```

    For the standard VST3 directory:

    ```bash
    yabridgectl add "$HOME/.wine/drive_c/Program Files/Common Files/VST3"
    ```

    For the standard CLAP directory:

    ```bash
    yabridgectl add "$HOME/.wine/drive_c/Program Files/Common Files/CLAP"
    ```

    <Note>
      The VST2 directory may be capitalized as `VSTPlugins` on your system. Some plugins also install to a directory directly under `Program Files` rather than inside `Steinberg`. Add those directories separately with `yabridgectl add` if needed.
    </Note>

    You can see all currently managed directories at any time:

    ```bash
    yabridgectl list
    ```
  </Step>

  <Step title="Run yabridgectl sync">
    Run `yabridgectl sync` to set up yabridge for all plugins in your managed directories:

    ```bash
    yabridgectl sync
    ```

    This command creates the bridged plugin files in your Linux plugin directories:

    - VST2 plugins → `~/.vst/yabridge`
    - VST3 plugins → `~/.vst3/yabridge`
    - CLAP plugins → `~/.clap/yabridge`

    Re-run `yabridgectl sync` whenever you install or remove a Windows plugin to keep everything up to date.

    <Tip>
      Run `yabridgectl status` to get an overview of all managed plugin directories and the installation status of each plugin. This is useful for spotting plugins that need a sync or checking that everything is set up correctly.
    </Tip>
  </Step>

  <Step title="Configure your DAW to scan the output directories">
    Add the following directories to your DAW's plugin search paths:

    - **VST2**: `~/.vst` (or the full path `$HOME/.vst`)
    - **VST3**: `~/.vst3` (or the full path `$HOME/.vst3`)
    - **CLAP**: `~/.clap` (or the full path `$HOME/.clap`)

    Then trigger a plugin rescan in your DAW. Your Windows plugins should appear as native Linux plugins.

    <Warning>
      Make sure you are using native packages of your DAW (.deb, .rpm, AUR, etc.) and not a Flatpak. yabridge does not work with Flatpak DAWs.
    </Warning>
  </Step>
</Steps>
