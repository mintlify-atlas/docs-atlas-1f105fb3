yabridge seamlessly supports using both 32-bit and 64-bit Windows VST2, VST3, and CLAP plugins in 64-bit Linux plugin hosts as if they were native plugins. Its modern concurrent architecture and focus on transparency allows yabridge to be both fast and highly compatible, while also staying easy to debug and maintain.

## Plugin formats

yabridge bridges all three major Windows plugin formats into their Linux equivalents:

| Format | Windows plugin | Linux output location |
|--------|----------------|-----------------------|
| VST2   | `.dll`         | `~/.vst/yabridge`     |
| VST3   | `.vst3`        | `~/.vst3/yabridge`    |
| CLAP   | `.clap`        | `~/.clap/yabridge`    |

### 32-bit support via bitbridge

yabridge can also load 32-bit Windows plugins in your 64-bit Linux DAW. It automatically detects whether a plugin is 32-bit or 64-bit on startup and handles it accordingly. If you installed yabridge from a distro package, your distro may have disabled this feature — check your package's documentation.

<Note>
  The bitbridge is not compatible with Wine's WoW64 build mode. If your Wine installation does not ship a `wine64` binary, the bitbridge will likely not work.
</Note>

## Architecture overview

yabridge consists of two main components:

- **Linux plugin library** (`libyabridge-{vst2,vst3,clap}.so`) — loaded by your DAW as a native plugin. It communicates with the Wine host over Unix domain sockets and exchanges audio data via shared memory buffers.
- **Wine plugin host** (`yabridge-host.exe`) — a Winelib application that loads and runs the actual Windows plugin `.dll` or `.vst3` bundle inside Wine.

This split design means your DAW never interacts with Wine directly. All communication is asynchronous and concurrent, keeping audio processing on its own thread separate from GUI and event handling.
